# Preferences

Application-wide preferences live in `QSettings` under `preferences/*`. Open the dialog with **Edit → Preferences…** (`Ctrl+,`).

The dialog has four groups: **Graphics**, **Interface**, **Reliability**, **Performance**. Graphics and Interface require an app restart (the GPU context isn't hot-swappable; `QT_SCALE_FACTOR` is read only at `QApplication` construction). Reliability and Performance apply at the next pipeline start.

## Graphics — GPU profiles

GPU selection is modeled as a list of named **profiles**. Each profile maps a stable id to a physical device UUID. The special **Default** profile (id `"default"`) is always present, cannot be removed, and its name is fixed — it's what plugins fall back to when they aren't pinned to a specific profile.

### Default profile keys

Legacy single-GPU keys, preserved so existing installs migrate with zero effort:

| Key | Type | Description |
|-----|------|-------------|
| `preferences/gpu/device_uuid` | `QByteArray` (16 bytes) | Default profile's adapter UUID. |
| `preferences/gpu/device_index` | `int` | Default profile's index fallback. `-1` = Auto (`discrete > integrated > first`). |

### Named profiles

Stored as a `QSettings::beginWriteArray("preferences/gpu/profiles")`:

| Key | Type | Description |
|-----|------|-------------|
| `id` | `QString` | Stable autogenerated `QUuid` (no braces). |
| `name` | `QString` | User-editable display name. |
| `device_uuid` | `QByteArray` (16 bytes) | Selected device UUID; empty = Auto. |
| `device_index` | `int` | Human-readable fallback hint for cross-machine loads. |

### Vulkan loader layers

| Key | Type | Default | Description |
|-----|------|---------|-------------|
| `preferences/gpu/disable_amd_switchable` | bool | `true` | Disables `VK_LAYER_AMD_switchable_graphics` (caps Vulkan at 1.1 on AMD hybrid laptops). |
| `preferences/gpu/extra_disabled_layers` | `QString` | — | Comma-separated list of extra layers to disable. Joined with the above into `VK_LOADER_LAYERS_DISABLE`. |
| `preferences/gpu/vulkan_validation` | bool | `false` | Enable `VK_LAYER_KHRONOS_validation` (requires the Vulkan SDK installed locally). |

## Interface

| Key | Type | Default | Description |
|-----|------|---------|-------------|
| `preferences/ui/scale_factor` | double | `1.0` | UI scale. Discrete presets: `0.75`, `1.0`, `1.25`, `1.50`, `1.75`, `2.0`. Multiplies Qt's per-monitor DPI scaling. **Restart required** — applied via `qputenv("QT_SCALE_FACTOR", …)` before `QApplication` construction. |
| `preferences/ui/zoom_fit_on_open` | bool | `false` | If true, every project open zooms / pans the Pipeline view so all nodes fit (same as the ⛶ button). Applies immediately, no restart. |
| `preferences/ui/node_stats_interval_ms` | int | `500` | Refresh period for the per-node stats badge (FPS / latency / drops / health) under each pipeline node. Combo presets: Off / 500 ms / 1 s / 2 s / 5 s (`0` = badges hidden, timer stopped). Engine measurement window (`kLatencyWindowSeconds`) is unaffected — only the UI poll cadence. Applies immediately, no restart. |

## Reliability — node watchdog

Maps 1:1 onto `spc::NodeWatchdog::Config`. Timeouts are stored in **milliseconds**; the engine converts to microseconds internally.

| Key | Type | Default | Description |
|-----|------|---------|-------------|
| `preferences/engine/watchdog/default_hang_timeout_ms` | int | `10000` | Time before a non-source node is considered hung. |
| `preferences/engine/watchdog/source_hang_timeout_ms` | int | `30000` | Same, for source nodes (file demuxers / cameras start slowly). |
| `preferences/engine/watchdog/gpu_hang_timeout_ms` | int | `30000` | Same, for GPU-routed nodes (FFmpeg + Vulkan teardown is slow). |
| `preferences/engine/watchdog/poll_interval_ms` | int | `500` | Watchdog tick rate. Fast-path `DEVICE_LOST` recovery rides this interval. |
| `preferences/engine/watchdog/max_restarts` | int | `3` | Max restart attempts per node before the watchdog disables it and cascade-disables downstream. `0` disables on first hang. |
| `preferences/engine/watchdog/restart_cooldown_ms` | int | `5000` | Minimum gap between restart attempts. |
| `preferences/engine/watchdog/auto_restart` | bool | `true` | If false, the watchdog detects but doesn't restart. |

## Performance

| Key | Type | Default | Description |
|-----|------|---------|-------------|
| `preferences/engine/pool/frame_count` | int | `32` | Pre-allocated frame pool size. Clamped by the engine to `[4, 64]`. Larger = more concurrency, more memory. |

## Apply model

- **Graphics / Interface** → app restart required. The dialog offers **Restart Now** / **Later** / **Cancel** when one of these is changed.
- **Reliability / Performance** → applied at the next `PipelineEngine::start()`. `MainWindow::apply_engine_preferences()` reads QSettings and pushes the values to `engine_->set_watchdog_config()` and `engine_->set_pool_frame_count()` at the top of `on_play_clicked()` and `restart_pipeline()`. If the pipeline is currently running, Stop and Start to pick up new values.
- Engine-section changes are **committed to QSettings independently** of the restart prompt — cancelling the restart prompt does not discard them.

## Environment overrides

A handful of preferences also have environment-variable equivalents that the CLI runner uses (the CLI doesn't read the GUI's QSettings store):

| Variable | Replaces |
|----------|----------|
| `SPC_GPU_DEVICE_INDEX` | `preferences/gpu/device_index` for the Default profile. |
| `VK_LOADER_LAYERS_DISABLE` | `preferences/gpu/disable_amd_switchable` + `extra_disabled_layers` (joined). |
| `SPC_VULKAN_VALIDATION` | `preferences/gpu/vulkan_validation` when the GUI hasn't called `set_validation_enabled` (CLI path). |

See [cli.md](cli.md) for full CLI environment.
